### 2.1 Harden CameraService: resolution fallback chain ✅

**Description**: Modify `CameraService.start()` to use a resolution fallback chain instead of a single `getUserMedia` call. Try 1280x960 -> 960x720 -> 640x480 -> unconstrained (facingMode only). Also set `playsinline`, `muted`, and `autoplay` attributes programmatically on the video element.

**Files to modify**:
- `frontend/src/app/core/services/camera.service.ts` -- refactor `start()` method

**Current state**: Single `getUserMedia({ video: { width: { ideal: width }, height: { ideal: height }, facingMode } })` call that fails entirely on constraint error.

**Target state**: Loop through `RESOLUTION_CHAIN` array, catch `OverconstrainedError` on each, fall through to next. Final attempt uses `{ video: { facingMode } }` with no resolution constraint.

**Acceptance criteria**:
- `start()` attempts up to 4 different resolution configs before failing
- If first resolution throws `OverconstrainedError`, second is attempted
- If all 4 fail, `error` signal is set with descriptive message
- `playsinline`, `muted`, `autoplay` attributes are set on video element before `play()`
- Existing callers (KioskComponent) do not need code changes
- CameraConfig interface now includes `maxCaptureWidth` and `jpegQuality` (from design)

### 2.2 Harden CameraService: canvas scaling and JPEG quality cap ✅

**Description**: Modify `captureFrame()` to scale down the canvas when video resolution exceeds `maxCaptureWidth` (1280px default). Apply platform-appropriate JPEG quality (0.7 mobile, 0.8 desktop) via `PlatformService.jpegQuality()`. This ensures each frame is ~100-200KB instead of 2-4MB on 4K phone cameras.

**Files to modify**:
- `frontend/src/app/core/services/camera.service.ts` -- refactor `captureFrame()` method

**Current state**: `canvas.width = videoElement.videoWidth; canvas.height = videoElement.videoHeight; toDataURL('image/jpeg', 0.8)` -- full resolution, fixed quality.

**Target state**: If `videoWidth > maxCaptureWidth`, calculate scaled dimensions maintaining aspect ratio. Canvas uses scaled dimensions. `toDataURL` uses `PlatformService.jpegQuality()`.

**Acceptance criteria**:
- On a video with `videoWidth: 3840, videoHeight: 2160`, canvas is created at `1280x720`
- On a video with `videoWidth: 640, videoHeight: 480`, canvas stays at `640x480` (no upscaling)
- JPEG quality is 0.7 when `PlatformService.isNative()` or `isTablet()` is true
- JPEG quality is 0.8 when on desktop browser
- Resulting base64 string for a 1280px frame at 0.7 quality is verifiably smaller than before

**Dependencies**: 2.1 (CameraConfig includes maxCaptureWidth/jpegQuality), 1.3 (PlatformService)
**Priority**: CRITICAL
**Effort**: S
